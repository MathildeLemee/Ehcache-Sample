Getting started with Ehcache

Last release of Ehcache is 2.6.0. Release notes.

What is Ehcache?

Ehcache is a distributed in-memory cache solution, aimed at scalability.
The key goal is to improve performance by storing data in memory so that subsequential access to this data will be faster.
By allowing the use of terabytes of memory, it is able to cache and replicate large amount of data with a low latency.

You can download the distribution here.

How do I use it?

import net.sf.ehcache.*;

// Create the cache manager and add a cache to it
CacheManager cacheManager = CacheManager.create();
cacheManager.addCache("myCache");
Cache cache = manager.getCache("myCache");

// Put an Element in the cache
cache.put(new Element("key", "value");

// Get the Value for the Key in the cache
Element element = cache.get("key");
System.out.println(element.getValue());

// Delete the Element from the cache
cache.remove("key");

// Shutdown the cachemanager
cacheManager.shutdown();


Enable BigMemory and unleash the power

Garbage Collection is the reason why most applications can't scale.
BigMemory allows Ehcache to use Terabytes of memory without having Garbage Collection problems.
And it is very easy to enable it when using Ehcache.

import net.sf.ehcache.*;

// Create the cache manager and add a cache with 10 Gigabytes of BigMemory
Configuration configuration = new Configuration()
    .cache(new CacheConfiguration().name("bigCache")
        .maxBytesLocalOffHeap(10, MemoryUnit.GIGABYTES)
    );

CacheManager cacheManager = CacheManager.create(configuration);
Cache cache = manager.getCache("bigCache");
